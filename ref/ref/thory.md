## 포인터 & 레퍼런스

# 포인터

**주소를 저장하는 변수**

```cpp
int a = 10;
int* p = &a;
```
a 현재 값 10 -> p에 a주소를 저장
- &a 는 a의 주소 의미
- p 는 그 주소를 저장 하는 변수
- 즉 p는 a를 가르킨다

# 레퍼런스

```cpp
int a = 10;
int& r = a;
```

r은 a와 완전히 동일한 존재(r은 a의 별명 이라 생각)

# 포인터 메모리 특징

```cpp
int a = 10;
int* p = &a;
```

p -> 주소(위치)
*p -> 그 주소에 있는 값

예시를 들면 *p = 20이란 코드가 저기에 추가됐다 그럼 p는 a를 의미
a를 20으로 변경 해라 라는 말이다

**p vs *p**
- p = &b -> 주소 변경
- *p = 50 -> 값 변경

# 레퍼런스 특징

int& r = a;
- r은 a의 별명
-**절대" 다른 변수로 변경 불가**

r = b;
a = b(값 복사)

레퍼런스는 포인트와 다르게 대상을 바꾸는 기능이 없고 값만 바꿀 수 있다

```cpp
int a = 5;
int b = 100;

int& r = a;
r = b;
b = 200;
```
int& r = a;를 선언한 순간 r은 평생 a만 본다 무슨 말이냐면 코드를 보자
일단 a는 5 b는 100을 선언 했다 그리고 int& r = a; 레퍼런스 선언 했고
저걸 선언 한 순간 부턴 r은 a만 본다 r = a를 의미한다
근데 b값을 r에 복사했다 방금 말했듯이 r은 a만 본다 그럼 a 값이 100
으로 바뀌겠지 b는 200으로 바꿈 그럼 b는 200으로 바뀔거고 그럼 출력
값이 a는 100 b는 200이 될 것이다

# 왜 쓰는가?
레퍼런스
- 대상이 절대 변하지 않을 때
- 코드가 깔끔 해진다

포인터
- 대상이 바뀔 수 있을 때
- 없을 수도 있을 때(nullptr)

# 포인터 복사

```cpp
int* p = &a;
int* q = p;
```
q는 p의 값을 복사(즉 주소 복사)
p -> a
q -> a

**하지만 p를 바꿔도 q는 안 바뀐다**

p = b&; 이렇게 해도
p -> b
q -> a
이렇게 된다는 말이다 p랑 q는 아예 다른 변수이다
저건 p의 주소만 복사 한걸 의미

# 레퍼런스 + 포인터 함께 쓰는 경우

```cpp
int a = 10;

int& r = a;
int* p = &a;

r = 20;
*p = 30;
r = 40;
```
이거 자세히 코드 보면
r = a
p -> a
둘다 a를 가르키고 있음 그럼 둘다 a값인데 어떤 값이 출력이 될까?

마지막에 대입된 값이 최종 결과이다
r = 20 -> a = 20
*p = 30 -> a = 30
r = 40 -> a = 40
결국 a는 40 계속 덮어진다

# 결론
포인터는 변수의 주소를 저장하여 간접적으로 값을 수정 할 수 있고,
가리키는 대상을 변경 할 수 있다
반면 레퍼런스는 변수의 별명으로, 초기화 이후 다른 대상을 참조할 수
없으며 항상 같은 원본을 직접 수정한다
